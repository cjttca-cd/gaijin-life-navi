## 1. 製品概要

### 1.1 ワンライナー
**在日外国人が日本での生活手続きを AI と対話しながら解決できる多言語プラットフォーム**

### 1.2 ビジョン
在日外国人396万人が「日本で暮らすことの不安」をゼロにする。
Reddit r/japanlife に書き込む代わりに、このアプリに聞けば即座に正確な答えが返ってくる世界。

### 1.3 USP（Unique Selling Proposition）
1. **AI 対話式**: 静的ガイドではなく、個人の状況に合わせたパーソナライズ回答
2. **多言語ネイティブ**: 14言語対応（AI動的翻訳、テンプレ翻訳ではない）
3. **手続き完結型**: 情報提供だけでなく「次に何をすべきか」のアクション提示
4. **ワンストップ**: 求人サイト、翻訳アプリ、Q&A掲示板が分断している現状を統合

## 3. コア機能の詳細設計

### 3.1 AI Life Concierge（コアエンジン）

**概要**: 多言語 AI チャットボット。ユーザーの質問に対話形式で回答し、具体的なアクション（手続きの手順、必要書類、窓口情報）を提示する。

**技術設計**:
```
ユーザー入力（任意言語）
  ↓
言語検出 + 翻訳（必要に応じて）
  ↓
意図分析（カテゴリ: 銀行/ビザ/医療/住居/その他）
  ↓
RAG検索（ナレッジベース: ISAポータル、入管庁、各省庁の公式情報）
  ↓
LLM生成（Claude/GPT — 文脈+検索結果+ユーザープロフィール）
  ↓
回答生成 + ソース引用 + 次のアクション提示
  ↓
ユーザー言語に翻訳して出力
```

**RAG ナレッジベース構成**:

| ソース | 内容 | 更新頻度 |
|--------|------|---------|
| 外国人生活支援ポータル (ISA) | 行政手続き全般、17言語 | 月次 |
| 出入国在留管理庁 | ビザ・在留資格情報 | 法改正時 |
| 各自治体の多文化共生情報 | 地域固有の窓口・サービス | 四半期 |
| 金融庁・全銀協 | 銀行口座開設の要件 | 半年 |
| 厚生労働省 | 国保・年金・労働法 | 法改正時 |
| r/japanlife FAQ | コミュニティの実体験知識 | 月次 |

**差別化ポイント**:
- 汎用 ChatGPT との違い: 日本の行政手続きに特化した RAG + ユーザープロフィール（在留資格、国籍、地域）に応じたパーソナライズ
- 既存アプリとの違い: 「読む」のではなく「聞く」体験。会話の中で段階的に情報を提供

**免責事項**: 全ての回答に「一般的な情報提供であり、法的助言ではありません。最新情報は関係機関にご確認ください」を表示。ソースURLを必ず引用。

### 3.2 Banking Navigator

**概要**: 銀行口座開設を段階的にガイドするウィザード。

**フロー**:
```
STEP 1: 基本情報入力
  - 在留資格 / 在留期間 / 国籍 / 居住地域
  ↓
STEP 2: 銀行レコメンド
  - ユーザーの条件に合う銀行を推薦（多言語対応度、ATM利便性、手数料）
  - 各銀行の特徴比較表を表示
  ↓
STEP 3: 必要書類チェックリスト
  - 在留カード / パスポート / 住民票 / 印鑑 or サイン
  - 書類ごとの取得方法ガイド
  ↓
STEP 4: 手続きガイド
  - 窓口での会話テンプレート（日本語 + 母語）
  - よくある質問と回答（「ハンコが必要と言われたら？」）
  ↓
STEP 5: トラブルシューティング
  - 「口座開設を断られた場合」の対処法
  - 代替銀行の提案
```

**差別化**: 既存のどのサービスも「銀行口座開設」をここまで詳細にガイドしていない。Reddit での最頻出痛点に直接対応。

### 3.3 Visa & Status Navigator

**概要**: 在留資格に関する手続きを AI がパーソナライズガイド。

**対応手続き**:
| 手続き | 対象者 | 複雑度 | 期限 |
|--------|--------|--------|------|
| 在留期間更新 | 全員 | ★★★ | 在留期限3ヶ月前〜 |
| 在留資格変更 | 留学→就労 等 | ★★★★ | 状況により |
| 永住許可申請 | 10年以上在留 | ★★★★★ | なし |
| 再入国許可 | 出国予定者 | ★★ | 出国前 |
| 資格外活動許可 | 留学生（バイト） | ★★ | 活動前 |
| 家族滞在 | 家族呼び寄せ | ★★★★ | 状況により |

**免責設計**: 「情報提供」と「代行」の境界を厳格に管理。具体的な申請書作成・代理提出には踏み込まず、行政書士マッチングに誘導。

### 3.4 Medical Access Guide

**概要**: 医療受診を支援する機能。

**主要機能**:
1. **症状→病院検索**: 症状を母語で入力 → 適切な診療科を提案 → 近隣の病院検索（多言語対応有無で絞り込み）
2. **受診会話テンプレート**: 症状を日本語で説明するためのフレーズ集（音声再生付き）
3. **保険ガイド**: 国民健康保険 / 社会保険の仕組み説明、高額療養費制度の案内
4. **緊急時ガイド**: 救急車の呼び方（119）、救急病院検索

### 3.5 Document Scanner & Translator

**概要**: カメラで書類を撮影 → OCR → AI翻訳 + 内容説明。

**対応書類**:
- 住民票 / 在留カード / 年金通知書 / 国保通知 / 納税通知書
- 銀行からの通知 / 契約書 / 役所からの書類

**技術**: OCR (Tesseract or Cloud Vision API) → LLM で文脈解釈 → 母語に翻訳 + 「何をすべきか」の解説付き

### 3.6 Community Q&A

**概要**: ユーザー同士の情報共有プラットフォーム。

**設計**:
- 国籍別チャンネル（中国語 / ベトナム語 / 韓国語 / 英語 / ポルトガル語）
- トピック別カテゴリ（銀行 / ビザ / 医療 / 住居 / 仕事 / 日常生活）
- AI による自動翻訳（国籍を超えた情報共有）
- AI による回答品質チェック（誤情報のフラグ付け）
- 「ベストアンサー」機能 → ナレッジベースに還元

### 4.2 技術スタック

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | React Native (Expo) | iOS/Android 同時開発、Web 対応も可 |
| Web | Next.js 14 | SEO対応 LP + Web版アプリ |
| API Gateway | Cloudflare Workers | 低レイテンシ、グローバルエッジ |
| バックエンド（AI） | Python / FastAPI | LLM連携、RAG構築に最適 |
| バックエンド（App） | Node.js / Next.js API Routes | フロントと統合 |
| LLM | Claude API (Anthropic) | 多言語品質が高い、日本語に強い |
| RAG | Pinecone + LangChain | ベクトル検索の定番 |
| OCR | Cloud Vision API | 日本語OCR精度が高い |
| DB | PostgreSQL (Supabase) | 認証+DB+ストレージを一体提供 |
| ストレージ | Cloudflare R2 | 低コスト、S3互換 |
| 認証 | Supabase Auth | ソーシャルログイン対応 |
| 決済 | Stripe | 多通貨対応、サブスク管理 |
