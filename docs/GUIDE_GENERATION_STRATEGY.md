## 1. 内容模型 + 用户层级

> 更新: 2026-03-03 — 概要級廃止（Z+Strategist 合意）。AI Chat = 深度級のみ。
> AI Chat 設計詳細 → `tasks/ai-chat-tier-design.md`（v2）

### 1.1 两层内容

```
Layer 1: Guide（统一质量）
  ├── Free（8篇热门话题，Guest 可见）→ 引流样品 + inline AI 向导引流
  └── Registered（37篇，注册可见）→ "还有40篇同等质量的指南"
       ↓
Layer 2: AI Chat（深度級のみ）→ "你的情况具体该怎么办"（个性化 = 核心付费价值）
  └── Guest: 静态预览（深度級回答示例展示）→ 注册引导
```

### 1.2 三层用户 × 功能矩阵

| 功能 | 🔓 Guest | 🆓 Free (¥0) | ⭐ Standard (¥720) | 💎 Premium (¥1,360) |
|------|----------|-------------|-------------------|---------------------|
| Free Guide（8篇） | ✅ 全文 | ✅ 全文 | ✅ 全文 | ✅ 全文 |
| Registered Guide（37篇） | excerpt+注册CTA | ✅ 全文 | ✅ 全文 | ✅ 全文 |
| AI Chat（深度級） | ❌（静态预览のみ） | **5次/lifetime** | 300次/月 | 无限 |
| 従量チャージ（深度級） | — | ✅ | ✅ | — |
| Re-engagement（30日毎1次深度） | — | ✅（用完後） | — | — |
| Auto Tracker | ❌ | ✅ | ✅ | ✅ |
| 广告 | あり | あり | なし | なし |

### 1.3 AI Chat = 深度級（个性化，核心壁垒）

| 能力 | 深度級 |
|------|--------|
| 完整步骤+材料+注意事项 | ✅ |
| LLM旧知識修正 | ✅ |
| 对话式答疑 | ✅ |
| Profile 个性化（国籍/签证/地区） | ✅ |
| 具体机构名+评价 | ✅ |
| 判断决策树 | ✅ |
| 地域差异/跨域联动/web_search | ✅ |

> Guide = "怎么做"（引流 + 自助工具）
> AI 向導 = "你该怎么做"（个性化 = 付费价值核心）

### 1.4 注册时强制 Profile（3字段）

注册流程中必须收集：国籍 + 在留资格 + 居住地域。确保深度級首次体验即可个性化。

**关键原则**:
- **Guide 只有一种质量标准** — 全部写到"完整 how-to"级别（步骤+材料+注意事项）
- **Free 不是"概述"**，是"样品" — 用户看到的是完整的干货，产生"这网站不错"的信任
- **注册动机 = 数量 + AI 向导** — "注册后还有40篇" + "AI 向导给你个性化建议"
- AI Chat = 基于 Profile（国籍/签证/地区）的个性化建议 + knowledge 全量
- **Guide 不写全部 knowledge** → 具体机构评价/判断逻辑/地域差/踩坑细节只在 AI 层
- **Guest 看不到 AI 对话** → 静态预览展示价值，注册后才能使用

## 5. AI-only 层（不写入任何 Guide）

以下内容 **只存在于 knowledge/ 中，仅 AI Chat 可用**。这是产品核心壁垒。

### 5.1 AI-only 内容分类

| 类别 | 定义 | 示例 | 不入 Guide 的理由 |
|------|------|------|-------------------|
| **具体机构评价** | 对特定银行/医院/运营商的正面或负面评价 | "ゆうちょ窓口は電話番号不要で開設可" / "MUFG外国人は窓口のみ" | 法律风险 + 时效性 + 机构可能变更政策 |
| **地域差** | 同一制度在不同地区的执行差异 | "新宿区严格，涩谷区宽松" / "XX区の役所は英語対応あり" | 个性化价值 + 时效性极高 |
| **判断决策树** | 根据用户属性（国籍/签证/收入等）做出的条件分支判断 | "中国国籍留学生 → 租税条約全額免税" / "有信用卡 → Route A、没有 → Route B" | AI 核心差异化 — 这是用户付费的理由 |
| **具体金额/费率** | 随时可能变动的数字 | "ゆうちょ ATM 手数料 110円" / "楽天モバイル 最低月額 1,078円" | 时效性 → AI 用 web_search 实时查 |
| **踩坑细节** | 来源于个人体验的具体踩坑描述 | "e-Tax で IPv6 接続不可、モバイル回線で回避" | 太细碎，Guide 写了用户不消化，对话中按需提供更有效 |
| **跨域联动建议** | 涉及多个域的组合建议 | "出生届当天也办児童手当+健保" / "银行开户和手机号の循環依存解法" | AI 的跨域能力 — Guide 是单域的 |
| **🔒 标记内容** | knowledge 中 `> 🔒 AI-only:` 明确标记的 | "知人の番号を借りる方法" | 敏感信息，仅在个别对话中酌情使用 |
| **LLM旧知識の修正详情** | `⚠️ LLM旧知識修正` 的具体比对内容 | "LLM说楽天クレカ必須 → 実際は口座振替もデビットも対応" | Guide 直接写正确信息即可，不需要解释"LLM哪里错了" |

### 5.2 为什么 AI-only 层是核心壁垒

```
ChatGPT / 其他 AI:
  - 通用知识（可能过时）
  - 不知道具体机构差异
  - 不知道用户的国籍/签证/地区
  - 无法做条件分支判断
  = 只能给"一般论"

我们的 AI Chat:
  - knowledge/（经过验证的经验库）
  - 用户 Profile（国籍/签证/地区）
  - 判断决策树（条件分支推荐）
  - web_search（实时金额/费率）
  = 给出"你该怎么办"的个性化答案
```

**→ Guide 是让用户知道我们有这个能力的"样品"；AI Chat 才是真正的产品。**

## 6. Guide 生成方法论（完整体系）

> **本节是 Guide 生成的权威规范**。svc-* AGENTS.md 中的 Guide 规则从本节派生。
> 最后更新: 2026-03-01（基于 8 篇 Free Guide 首轮审阅反馈重构）

### 6.1 核心架构

```
knowledge/*.md（Agent 内部参考）
       ↓ [生成规则]
guides/*.{lang}.md（用户向けコンテンツ）
       ↓ [API 配信]
Navigator 界面（用户看到的）
```

**关键原则**:
- knowledge/ = 经验库（Agent 专用、不公开、含判断逻辑和踩坑细节）
- guides/ = 用户向けコンテンツ（结构化、可读性优先、access control）
- Guide **由 svc-* agent 自己生成**（不由 main agent 代写）——agent 有域专有 AGENTS.md、knowledge、compliance rules

### 6.2 文件结构

```
svc-{domain}/workspace/guides/
├── {slug}.zh.md      # 中文版
├── {slug}.ja.md      # 日文版
├── {slug}.en.md      # 英文版（Phase 3c）
├── {slug}.vi.md      # 越南语版（Phase 3c）
└── {slug}.pt.md      # 葡萄牙语版（Phase 3c）
```

### 6.3 Frontmatter 格式

```yaml
---
title: "外国人银行开户指南"
slug: bank-account-opening
domain: finance
access: free          # free | registered
lang: zh
tags: [银行, 开户, 外国人]
source_knowledge:     # 生成来源（限定信息源）
  - account-opening.md
  - banks-overview.md
last_generated: 2026-03-01
version: 1
---
```

### 6.4 信息来源规则（⚠️ 最重要）

Guide 的信息分为两类，处理方式不同：

| 信息类型 | 来源 | 处理方式 |
|---------|------|---------|
| **背景说明**（制度是什么、为什么重要） | LLM 通用知识 + knowledge/ | 可使用 LLM 知识作为背景铺垫，但必须与 knowledge 一致 |
| **核心内容**（操作步骤、材料清单、注意事项、时间线） | **仅 knowledge/** | 必须且只能来自 frontmatter source_knowledge 指定的文件 |

**具体规则**:

1. **source_knowledge 限定**: Guide 的核心内容**仅来自** frontmatter 中 `source_knowledge` 指定的 knowledge 文件
2. **LLM 知识允许用于背景**: 用户需要基础信息才能理解进阶内容，所以"日本的健康保险是什么""确定申告是什么"等背景说明可以用 LLM 知识
3. **LLM旧知識修正 优先**: knowledge 中标记 `⚠️ LLM旧知識修正` 的内容，Guide **必须使用 knowledge 版本**，不得使用 LLM 默认知识（例：保険証→マイナ保険証、103万→123万）
4. **knowledge 不存在的信息 = Guide 不写**: 发现知识缺口时，不补充、不推测，在 announce 中说明缺口，请求 knowledge 补充
5. **Agent 専用 / AI-only 内容禁止泄露**: knowledge 中 `> 🔒 AI-only:` 标记的内容、具体机构评价、判断决策树等，**不得出现在 Guide 中**

### 6.5 Guide 与 AI-only 的内容边界（⚠️ 核心 — Agent 必读）

> Guide 的本质是**引流样品** + **自助工具**。AI Chat 才是产品核心。
> Guide 质量统一（Free 和 Registered 完全相同），区别只是可见性。
> **切分维度不是深度，是话题热度**。

#### Guide 统一质量标准

所有 Guide（无论 Free 还是 Registered）都写到这种程度：

| 维度 | Guide（统一标准） | AI-only |
|------|------------------|---------|
| **信息深度** | "怎么做"（完整步骤+材料+注意事项） | "你该怎么做"（个性化） |
| **注意事项** | 完整的常见陷阱（概括性） | 具体案例 + 判断逻辑 |
| **机构/地域** | 可提及类别（"大手银行""格安SIM"），不评价具体机构 | 具体机构名 + 评价 + 地域差 |
| **数字/金额** | 概括性范围（"数千〜数万"）或不写 | 实时查询具体金额 |
| **判断决策树** | ❌ | ✅ |
| **个性化** | ❌ | ✅（基于 Profile） |
| **跨域关联** | ⚠️ "与XX相关，详见XX指南" | ✅ 具体联动建议 |
| **长度** | 1000-2000 字 | 无限制（对话） |

- 详细规则 → skill §2.3

### 6.7 措辞精度规则

1. **禁止无限定的绝对化表述**: 
   - ❌ "必须去诊所" / "不是在诊所取药" / "所有银行都需要手机号"
   - ✅ "**一般情况下**先去诊所" / "**通常**不在诊所取药（部分诊所有院内処方）" / "**大部分**银行需要手机号"
2. **数字/金额要标注时效**: 
   - ❌ "103万の壁"
   - ✅ "103万の壁（令和7年から123万に変更）"
3. **制度变更要明确时间节点**:
   - ❌ "保険証を持っていく"
   - ✅ "マイナ保険証 or 資格確認書（2024年12月の制度改正で従来の保険証は廃止）"
4. **地方政府机构统一用"役所"**: 不写"区役所"、"市役所"、"市町村"等具体类型（不同地区行政单位不同）
5. **"英語サービス" → "外国語サービス"**: 不限定为英语（"外語"も不可、必ず"外国語"）

### 6.10 自检步骤（Agent 生成后必须执行）

Agent 生成 Guide 后，在 announce/输出前，必须完成以下自检:

| # | 检查项 | 通过条件 |
|---|--------|---------|
| 1 | **事实核验（双向）** | (a) knowledge→Guide: source_knowledge 核心内容已反映；(b) **Guide→knowledge（逆向・必须）**: Guide 内每一条事实性描述都能在 source_knowledge 中找到根据。**找不到根据的 = hallucination → 删除** |
| 1.5 | **具体机构名检查** | Guide 中不得出现具体机构固有名词（银行名、医院名等）。§6.5 规则："可提及类别（大手银行、格安SIM），不评价具体机构"。具体机构名 = AI-only |
| 2 | **LLM旧知識** | knowledge 中所有 `⚠️ LLM旧知識修正` 标记的内容已正确反映 |
| 3 | **AI-only 泄露** | 无 `> 🔒 AI-only:` 内容、无具体机构评价、无判断决策树泄露到 Guide |
| 4 | **CTA 正确性** | Free = 注册+AI向導双轨（skill §2.1）/ Registered = AI向導（skill §2.2） |
| 5 | **免責声明** | tax/legal/medical/finance 域的 Guide 末尾有免責声明 |
| 6 | **措辞精度** | 无绝对化表述、数字有时效标注、制度变更有时间节点 |
| 7 | **长度** | 1000-2000 字（统一标准） |
| 8 | **Frontmatter** | title/slug/domain/access/lang/tags/source_knowledge/version 全部填写 |
| 9 | **Inline hook 配置** | 1-3 箇所の 💬 inline hook が適切な位置に（skill §2.3） |
| 10 | **タイトル整合性** | 各セクションがタイトルの約束に応えている |

**自检未通过 = 不得输出**。修正后重新自检。

### 6.11 多语言规则

- **独立生成，非翻译**: 各语言版本根据同一 knowledge 独立生成，适应目标语言用户的文化背景和表达习惯
- **优先级**: zh + ja 先行 → en → vi/pt（Phase 3c）
- **同一 knowledge 基础**: 所有语言版本的事实内容必须一致（来源相同），只是表达方式不同
- **CTA 和免責声明**: 各语言版本使用对应语言的 CTA 和免責文案

### 6.12 维护策略

- **单一真理源**: Z 只编辑 knowledge/*.md。Guide 从 knowledge 派生
- **knowledge 更新 → Guide 再生成**: knowledge 有实质性变更后，相关 Guide 需要再生成（版本号 +1）
- **Agent 不读 guides/**: Agent 只读 knowledge/（guides 是 Agent 自己写的，已知内容）
- **版本控制**: frontmatter version 字段追踪。每次再生成 version +1，last_generated 更新为当天日期
- **Z 审阅 Gate**: 生成后由 Z 审阅确认，通过后才算正式版本
