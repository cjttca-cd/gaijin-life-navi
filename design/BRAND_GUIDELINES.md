## 1. Brand Tone — Visual Expression

### 1.1 ブランドトーンの 5 軸

strategy/product-spec.md §4 で定義された 5 つのブランドトーンを、具体的なビジュアル表現に落とし込む。

#### 🔵 Trustworthy (信頼できる)

**言語的表現**: 「確認済みの情報に基づいています」「〜によると」

**視覚的表現**:
- **Primary Blue (`#2563EB`)** を基調色として一貫使用
- クリーンなホワイトスペースを十分に確保（密度を上げすぎない）
- 整列されたレイアウトグリッド（不揃いな配置は避ける）
- ソース引用は必ず表示し、出典の透明性を視覚化
- 数値やデータはカード内にきちんと構造化して表示
- 参考: Wise の「金融プロダクトとしての佇まい」— 派手さではなく整然とした信頼感

#### 🟢 Approachable (親しみやすい)

**言語的表現**: 「心配いりません」「一緒にやりましょう」

**視覚的表現**:
- **角丸 (`radiusSm` 8dp 以上)** を全コンポーネントに適用。角張ったシャープさは避ける
- **Secondary Teal (`#0D9488`)** と **Tertiary Amber (`#F59E0B`)** でブルーの冷たさを中和
- ドメインアイコンにカラフルな Container 背景（§1.7 Domain Accent Colors）を使い、視覚的な多様性を持たせる
- AI Chat のバブルはメッセンジャーアプリのような親しみやすい形状
- 参考: Duolingo の「楽しさ」のエッセンス — ただし「ゲーム的」にはしない（§1.2 避ける印象）

#### 💪 Empowering (エンパワーメント)

**言語的表現**: 「次のステップ」「あなたはこれができます」

**視覚的表現**:
- AI 回答に「次のアクション」を明確なカード/チップで提示
- Tracker のチェックボックス完了時にポジティブなフィードバック（`colorSuccess` + チェックアニメーション）
- 進捗を可視化（Tracker の完了率、残り Chat 回数）
- CTA は具体的なアクション動詞（「始める」「質問する」「確認する」）
- 参考: Wise の「自分で国際送金ができる」感覚 — ユーザーが主体

#### 🌏 Inclusive (多文化共生)

**言語的表現**: 「あなたの言語で」「どの国籍でも」

**視覚的表現**:
- 5 言語切り替えを常にアクセス可能に（Settings から 2 タップ以内）
- 人物イラスト/アイコンは抽象的な表現（特定の肌色・髪型を避ける）— §5 参照
- 国旗は言語選択画面のみで使用（ナショナリズムの印象を避ける）
- 色覚多様性を考慮（赤/緑の組み合わせだけで情報を伝えない — アイコン/テキストを併用）
- 参考: Google Maps の「国籍を問わない普遍的 UI」

#### 👔 Professional (プロフェッショナル)

**言語的表現**: 「正確な情報」「公式ソースに基づく」

**視覚的表現**:
- **Typography の階層を厳守** — 見出し/本文/キャプションの区別を明確に
- 情報密度は「読みやすさ」を最優先（詰め込まない）
- アニメーションは控えめ（300ms 以下、機能的な目的のみ）
- 過度な装飾・グラデーション・3D 効果は使用しない
- フォントウェイトは Regular (400) 〜 Bold (700) の範囲内。Ultra Bold (800+) や Light (300) は不使用
- 参考: Wise と Google Maps の「プロフェッショナルだが堅苦しくない」バランス

### 1.2 避けるべき印象（Anti-patterns）

| 避ける印象 | 具体例 | 理由 |
|-----------|--------|------|
| 上から目線 / お役所的 | 赤い警告バナーの多用、命令口調の UI テキスト、冷たいグレー主体の配色 | ターゲットユーザーは既に行政手続きに不安を感じている |
| 子供っぽい / ゲーム的 | マスコットキャラ、ポイントシステム、バッジコレクション、ピクセルアート | 金融・ビザ・医療は深刻なドメイン。軽く見える印象は信頼を損ねる |
| 民族ステレオタイプ | 国旗の乱用、特定国の伝統衣装、「日本=桜/富士山」の過度な使用 | 多国籍ユーザーベースへの配慮 |
| テック・スタートアップ感 | ダーク UI、ネオンカラー、グリッチエフェクト、サイバーパンク風 | ターゲットは技術者だけでない |
| 過度なミニマリズム | 情報量不足、グレー一色、アイコンのみでテキストなし | 多言語ユーザーには明示的なテキストが重要 |

## 2. Icon Style

### 2.1 方向性

- **スタイル**: Material Symbols Outlined (線画ベース)
- **線幅**: Weight 400 (default)、Grade 0、Optical Size 24
- **角丸**: Rounded variant を使用（Sharp は不使用）
- **塗り**: Outlined (非選択) / Filled (選択状態) の 2 段階

### 2.2 アイコン使用ルール

| ルール | 説明 |
|--------|------|
| **サイズの統一** | 20dp (small) / 24dp (standard) / 28dp (medium) / 40dp (large) の 4 段階のみ |
| **色の原則** | 基本は `colorOnSurface` (単色)。Active 状態のみ `colorPrimary`。ドメインアイコンはドメイン固有色 |
| **装飾禁止** | グラデーション、影、3D 効果をアイコンに適用しない |
| **ラベル併記** | アイコン単体では使用しない。必ずテキストラベルを併記する（BottomNav、アクションボタン等の例外あり） |
| **タップ領域** | アイコンサイズに関わらず最小 48dp × 48dp のタップ領域を確保 |

### 2.3 ドメインアイコンの表現

各 Navigator ドメインのアイコンは以下の形式で表示:

```
┌─────────────────┐
│   ┌────────┐    │
│   │  🏦    │    │  ← 48dp 丸型 Container (ドメイン Container 色)
│   │ (icon) │    │     内部: 28dp Material Icon (ドメイン Icon 色)
│   └────────┘    │
│    Banking      │  ← ドメイン名 (titleMedium)
│    6 guides     │  ← ガイド数 (bodySmall)
└─────────────────┘
```

### 2.4 カスタムアイコン

Material Symbols に存在しないアイコンが必要な場合:
- SVG で作成（線幅 1.5dp、24dp キャンバス）
- Material Symbols Outlined と視覚的に調和するスタイル
- `flutter_svg` パッケージで Flutter に組み込み
- カスタムアイコンは `assets/icons/` に配置

## 3. Illustration & Decoration

### 3.1 イラストレーション方針

**Phase 0**: イラストレーションは最小限に留める。AI が生成した具体的イラストは使用しない。

**使用する場面**:
- **オンボーディング (S06)**: ステップ説明の補助ビジュアル
- **空状態 (Empty State)**: Chat 履歴なし、Tracker 空の時
- **エラー状態**: ネットワークエラー、サーバーエラー

**スタイル方向**:
- **フラットイラスト**: 線画ベース、2〜3 色（Primary Blue + Teal + Amber のみ）
- **抽象的表現**: 人物は描くが顔の詳細は描かない（ジェンダー・人種を特定しない）
- **ジオメトリック**: 丸・角丸四角・曲線を基本形とした幾何学的な構成
- **サイズ**: 最大 200dp × 200dp（画面の 1/3 以下）

### 3.2 装飾ルール

| 要素 | ✅ 使用可 | ❌ 使用不可 |
|------|----------|------------|
| 背景パターン | 微妙なドットパターン (opacity 0.03) | 大胆なパターン、テクスチャ |
| グラデーション | Splash 画面のブルー系グラデーション | Rainbow、ネオン系 |
| セパレーター | 1dp ライン (`colorOutlineVariant`) | 装飾的なセパレーター（波線等） |
| 影 | Material 3 Elevation に準拠 | カスタムのハードシャドウ、ドロップシャドウ |
| 角丸 | `radiusSm` (8dp) 以上 | 角ばったシャープエッジ (radius 0) |
| アニメーション | Fade / Slide / Scale (300ms 以下) | Bounce 過多、回転、フラッシュ |

### 3.3 写真の使用

- **Phase 0**: 原則として写真は使用しない
- 将来使用する場合のルール:
  - 実際の日本の風景・場所の写真は OK（ストックフォトの「日本風」は NG）
  - 人物写真は多様性を確保（単一国籍・単一人種にしない）
  - フィルター: 彩度控えめ、明るめの自然光

## 4. Writing Tone & Microcopy

### 4.1 UI テキストのトーン

| トーン軸 | 方向 | 例 |
|---------|------|-----|
| フォーマル ↔ カジュアル | **中間〜ややカジュアル** | 「銀行口座を開設しましょう」（× 「銀行口座開設手続きを行ってください」） |
| 技術的 ↔ 平易 | **平易** | 「インターネットに接続できません」（× 「HTTP 接続エラーが発生しました」） |
| 受動的 ↔ 能動的 | **能動的** | 「質問してみる」（× 「質問が送信されます」） |

### 4.2 Microcopy ガイドライン

- **ボタン**: 動詞で始める（「登録する」「質問する」「始める」）
- **エラー**: 何が起きたか + どうすればいいか（「メールアドレスの形式が正しくありません。example@email.com のように入力してください」）
- **空状態**: 励ましのトーン（「まだ質問がありません。AI に何でも聞いてみましょう！」）
- **制限通知**: 共感 + 解決策（「今日の無料枠を使い切りました。アップグレードするとたくさん質問できます」）
- **免責事項**: 簡潔だが省略しない（「一般的な情報です。法的助言ではありません。」）

## 5. Multicultural Visual Rules

### 5.1 基本原則

**「誰も排除しない、誰も特別扱いしない」**

このアプリのユーザーは 190+ の国籍から来る可能性がある。特定の文化・国籍を想起させるビジュアルは、他のユーザーを疎外する。

### 5.2 具体的ルール

| カテゴリ | ルール |
|---------|--------|
| **国旗** | 言語選択画面 (S02) でのみ使用可。言語の識別目的に限定（🇺🇸 English, 🇨🇳 中文, 🇻🇳 Tiếng Việt, 🇰🇷 한국어, 🇧🇷 Português）。それ以外の画面では国旗を使用しない |
| **人物表現** | 抽象的なシルエットまたはジオメトリックアバター。肌色・髪色・顔立ちを特定しない。デフォルトアバターは `colorPrimary` 背景 + 白いシルエット |
| **文化的シンボル** | 桜、富士山、鳥居、招き猫などの日本文化シンボルは使用しない。アプリは「日本」のアプリではなく「日本に住む外国人」のアプリ |
| **宗教的シンボル** | 十字架、三日月、ダビデの星、卍などの宗教シンボルは使用しない（Medical の赤十字マークは国際的に認知されているため例外） |
| **食事の表現** | Food ドメインのアイコン/イラストでは特定の料理を描かない。汎用的な食器アイコン (`restaurant`) を使用 |
| **名前の例** | UI のプレースホルダーや例示に特定の国の名前を使わない。「Your name」「名前」等の汎用表現を使用 |
| **暦・日付** | グレゴリオ暦のみ使用。旧暦、和暦の表示は行わない（Japanese era 表記は行政 Navigator コンテンツ内のみ許可） |
| **色の文化的意味** | 赤=危険/エラー、緑=成功/安全のセマンティクスはグローバルに通用するため使用可。ただし色だけで情報を伝えず、アイコン/テキストを併記する |

### 5.3 言語切り替え UI

- 言語名は**各言語のネイティブ表記**で表示（English ではなく English、中文 ではなく 中文）
- 言語リストの順序: 端末ロケールの言語を最上部 → 残りはアルファベット順
- 言語選択後、全 UI が即座に切り替わること（ページリロード不要）

### 5.4 テキスト方向

- 全対応言語 (en, zh, vi, ko, pt) は LTR (Left-to-Right)
- Phase 1+ でアラビア語・ヘブライ語を追加する場合は RTL 対応が必要（現時点では考慮不要）

## 6. Do / Don't Examples

### 6.1 Color Usage

| ✅ Do | ❌ Don't |
|-------|---------|
| Primary Blue をメインアクションに限定使用。1 画面に Primary Button は原則 1 つ | Primary Blue をあらゆる場所に使い、何が重要か分からなくなる |
| Semantic Colors を用途通りに使う（Success=完了、Error=エラー、Warning=注意） | Error Red を「目立たせたい」だけの理由で非エラー要素に使う |

### 6.2 Typography

| ✅ Do | ❌ Don't |
|-------|---------|
| Type Scale の定義通りにサイズ・ウェイトを使用する。見出しは `headlineMedium`、本文は `bodyLarge` | 独自のフォントサイズ（例: 17sp）を発明する。Design System にないサイズを使わない |
| CJK テキストの行間を十分に確保し、漢字の密度に配慮する | Latin と同じ行間を CJK テキストに適用し、詰まった印象にする |

### 6.3 Spacing & Layout

| ✅ Do | ❌ Don't |
|-------|---------|
| Spacing System の定義値（4/8/12/16/24/32dp 等）のみを使用する | 中途半端な値（例: 10dp, 15dp, 22dp）を使って間隔がバラバラになる |
| 十分なホワイトスペースを確保し、情報の階層を視覚的に伝える | 画面にコンテンツを詰め込み、余白を「無駄なスペース」として排除する |

### 6.4 Icon & Illustration

| ✅ Do | ❌ Don't |
|-------|---------|
| Material Symbols Outlined で統一し、必要時のみカスタム SVG を追加する | 複数のアイコンセット（Material + FontAwesome + Feather等）を混在させる |
| ドメインアイコンは抽象的で普遍的な表現を使用する（例: Banking = 銀行建物アイコン） | ドメインアイコンに文化特有のシンボルを使う（例: Banking = 日本の五円玉） |

### 6.5 Cultural Sensitivity

| ✅ Do | ❌ Don't |
|-------|---------|
| ユーザーアバターは抽象的なジオメトリックデザインを使用する | デフォルトアバターに特定の肌色や髪型を設定する |
| 言語名をネイティブ表記で表示する（中文、Tiếng Việt） | 全てを英語表記にする（Chinese, Vietnamese）や、他言語での呼称を使う |

### 6.6 Component Usage

| ✅ Do | ❌ Don't |
|-------|---------|
| Emergency タブのアイコンを常に `colorError` (赤) で表示し、緊急性を視覚的に示す | Emergency を他のタブと同じ色で表示し、緊急時に見つけにくくする |
| AI Chat の免責事項を全ての AI レスポンスに表示する（位置はバブル外、控えめだが省略しない） | 免責事項を非表示にする、または最初の 1 回だけ表示する |

### 6.7 Tone of Voice

| ✅ Do | ❌ Don't |
|-------|---------|
| 「銀行口座を開設する方法をご案内します」— 対等で親切なトーン | 「以下の手続きに従い銀行口座を開設してください」— 命令・上から目線 |
| 「今日の無料枠を使い切りました。もっと質問するにはアップグレードがおすすめです」— 共感 + 提案 | 「制限に達しました。有料プランに加入してください」— 冷たい要求 |

### 6.8 Data Display

| ✅ Do | ❌ Don't |
|-------|---------|
| AI レスポンスにソース URL を明示し、情報の出典を透明にする | ソースを隠し、AI が独自に生成したかのように見せる |
| サブスクリプション料金を現地通貨 (JPY) で明確に表示し、全ての制限事項を事前に開示する | 料金を小さく表示し、制限事項を隠す |

## 8. Logo Guidelines

### 8.1 暫定ロゴ方針（Phase 0）

Phase 0 ではプロフェッショナルなロゴデザインは行わない。以下の暫定ルールで運用:

- **アイコン**: Material Symbols `explore` (Filled) をベースにした丸型アイコン
- **背景**: `colorPrimary` (`#2563EB`) の丸型
- **アイコン色**: `#FFFFFF`
- **サイズ**: 48dp (App Icon 内) / 80dp (Splash)
- **テキスト**: 「Gaijin Life Navi」— `titleLarge` (18sp, SemiBold)

### 8.2 Logo 使用ルール

- ロゴの周囲に最低 16dp のクリアスペースを確保
- ロゴの色は `colorPrimary` + White のみ（カラーバリエーション不可）
- ロゴの変形・回転・透明度変更は禁止
- 背景が暗い場合: White ロゴ。背景が明るい場合: `colorPrimary` ロゴ
